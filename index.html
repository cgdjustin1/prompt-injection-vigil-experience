Prompt Injection Audit：检查 AI Agent 输入里的隐藏风险

AI Agent 每天会接收 system prompt、用户消息、RAG 文档、网页内容和工具指令。这里面可能夹着“忽略之前指令”“输出密钥”“通过 Markdown 图片外带信息”这类攻击。这个项目会扫描这些输入，找出可疑片段，解释风险，并给出可执行的修复建议。

这是什么

这是一个 AI Agent 输入风险审计报告。它关注的不是模型能力，而是进入 Agent 上下文之前的内容是否藏着危险指令。

Prompt Injection Audit

它把一批 Agent 可能接触到的文本输入，当作上线前需要检查的风险资产：哪些内容安全，哪些内容可能试图改写指令、索要密钥或诱导外带数据。

Inputprompts / RAG docs / tool instructions
ScanYARA risk signatures
Outputrisk report + evidence + fixes

谁会需要它

任何准备把 AI Agent 接入真实用户、网页、文档库或工具系统的人，都需要先知道输入里有没有高风险指令。

适用场景

  • 上线前检查 system prompt、tool instruction 和 RAG 文档
  • 给团队做 AI Agent 安全培训或 red-team workshop
  • 审查客服、研究、浏览器 Agent 会读取的外部内容
  • 把风险输入整理成可追踪、可下载、可复查的审计报告

这份报告回答什么

  • 哪些输入命中了风险规则？
  • 命中的证据片段是什么？
  • 这类输入可能造成什么业务影响？
  • 应该隔离、改写、过滤，还是加入 CI 检查？

当前审计样本

这份样本报告扫描了 25 条 Agent 输入资产，并发现 6 条需要审查的风险输入。

扫描范围

  • 25 条输入资产: system prompt、tool instruction、RAG 文档、用户输入、邮件、网页片段
  • 风险类型: 指令绕过、密钥索取、Markdown 外带
  • 输出格式: HTML 报告、CSV 表格、JSON 原始结果
  • 使用方式: 用作上线前安全检查、团队评审或 workshop 样本

报告里有什么

  • 每条风险输入的 asset path、risk score 和命中规则
  • 触发规则的 evidence snippet
  • business impact：为什么这类输入危险
  • recommended actions 和 safer version：下一步怎么修

Methodology / Current Scope

  • 使用 cloned repo 的 Vigil YARA signatures
  • 本地离线扫描，不上传 prompt
  • 生成 JSON + CSV + 静态 HTML 审计报告
  • 报告包含 source provenance、score factors、evidence、business impact、safer version
  • 适合课堂、workshop、上线前 prompt sanity check

Limitations

  • 不是实时 LLM firewall
  • 尚未接入 transformer classifier / vector DB
  • YARA 规则不能覆盖所有 prompt injection
  • 结果应用作风险提示和审计起点，不是绝对安全判定
