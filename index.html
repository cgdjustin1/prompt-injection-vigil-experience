真实 Repo 证据

这版 v0 的重点不是功能多，而是纠正方向：一定要从真实 GitHub repo 出发。

这个体验适合什么场景？

课堂 / Workshop

不用部署复杂安全栈，直接让学生看到 prompt injection 如何被规则命中。

企业 AI 风险沟通

给业务方展示：Agent 上线前需要输入扫描、敏感信息外泄检测和审计层。

Builder 包装范式

证明一个开源安全 repo 可以被包成 30 秒可懂的体验，而不是丢给用户 README。

Boundary: v0 使用 Vigil 的 YARA signature 做离线扫描；还不是完整 Vigil REST API / transformer / vector DB 版本。下一版可以接真正后端，让用户输入任意 prompt 实时扫描。
